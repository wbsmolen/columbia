# Reporting Security Vulnerabilities

For a vulnerability in Columbia's own additions to this directory (the relay-auth check, the endpoint guard, or the rate limiter — see `VENDORED.md`), or in how Columbia deploys this gateway, follow the reporting process in the repository root's [`SECURITY.md`](../SECURITY.md).

For a vulnerability in the unmodified vendored upstream code itself, see [this page](https://www.cloudflare.com/.well-known/security.txt) for how to report it to Cloudflare.
